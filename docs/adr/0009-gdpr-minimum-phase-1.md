# ADR-0009: GDPR-минимум Phase 1 — consent, процессоры, удаление аккаунта

- **Статус:** принят
- **Дата:** 2026-07-03
- **Авторы:** Engineering (Phase 1 §7)

## Контекст

Аудит 04: нет cookie-consent, политика не раскрывает OpenAI/Resend, удаление аккаунта отсутствует, регион Supabase не задокументирован. Продажа физлицам в ЕС без этого — юридический блокер.

## Решение

1. **Cookie consent:** баннер с «только необходимые» / «принять все»; Vercel Analytics грузится только при `analytics: true` в `pharm_consent`.
2. **Политики:** privacy обновлена — процессоры Supabase (EU), Stripe, Vercel, Resend, OpenAI; отдельная страница `/cookies`.
3. **Удаление аккаунта:** server action `deleteAccount()` через Supabase Admin API; каскад profiles/addresses/favorites; orders → `user_id` NULL (налоговый срок).
4. **EU-регион:** `SUPABASE_REGION` / `NEXT_PUBLIC_SUPABASE_REGION` в env и политике; выбор региона при создании проекта — операционный чеклист, не код.
5. **Чат:** дисклеймер OpenAI в UI; отправка сообщения = инициатива пользователя.

## Последствия

- Проще: закрыт минимальный GDPR для запуска в Латвии; один переключатель домена через `lib/site.ts`.
- Сложнее: полный DSAR/экспорт — Phase 2; маркетинговые cookies пока нет.
- Затронут Playbook §1.5 (события), §15 (privacy).

## Альтернативы

- **Баннер «принять или уйти»** — отклонено: слишком агрессивно для e-commerce.
- **Удаление через email в support** — отклонено как единственный путь: не масштабируется, нарушает UX ст. 17.
