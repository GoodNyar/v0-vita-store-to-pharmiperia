# ADR-0008: Stripe Tax — PVN 21% (Латвия), цены включают НДС

- **Статус:** принят
- **Дата:** 2026-07-03
- **Авторы:** Engineering (Phase 1, roadmap §5)

## Контекст

Аудит готовности к продакшену (отчёт 04) фиксирует отсутствие НДС (PVN 21%) как юридический блокер. Публичная оферта и витрина заявляют, что цены указаны в евро и **включают PVN**. Заказ и вебхук сверяют `amount_total` с `total_cents`, но налог не рассчитывается и не сохраняется — reconciliation для бухгалтерии невозможен.

## Решение

1. **Stripe Tax** включён на `checkout.sessions.create` (`automatic_tax: { enabled: true }`), когда `STRIPE_TAX_ENABLED` не равен `false`.
2. **Цены tax-inclusive:** `tax_behavior: 'inclusive'` на всех `price_data` (товары и доставка); каталожные `Money` остаются брутто-суммами, клиент не видит «+21% сверху».
3. **Ставка:** стандартный PVN Латвии 21% через Stripe Tax (регистрация LV в Stripe Dashboard — обязательный шаг деплоя).
4. **Адрес для расчёта:** `shipping_details` передаётся из серверного заказа (курьер — полный адрес; пакоматы — адрес станции / LV по умолчанию).
5. **Хранение:** колонка `orders.tax_cents` — фактический НДС из `session.total_details.amount_tax` при fulfillment (fallback — формула gross×21/121).
6. **Сверка:** `amount_total` по-прежнему MUST совпадать с `total_cents` (брутто); расхождение tax без расхождения total — логируется, не блокирует оплату.

## Последствия

- Проще: юридически согласованные чеки Stripe, PVN в БД для отчётности, единая модель «цена на витрине = цена в Stripe».
- Сложнее: Stripe Tax MUST быть активирован в Dashboard (test/live); локально без Tax — `STRIPE_TAX_ENABLED=false`.
- Затронут Playbook §1.1 (сервер — источник истины), §1.7 (деньги в центах), ADR-0005 (fulfillment из вебхука).

## Альтернативы

- **Ручной расчёт 21% без Stripe Tax** — отклонено: нет налоговых чеков Stripe, выше риск ошибок при смене ставок/доставки.
- **Tax-exclusive цены (+21% в чекауте)** — отклонено: противоречит тексту оферты и текущей витрине.
- **Фиксированная ставка в коде без Stripe Tax API** — отклонено как основной механизм: не даёт compliance-отчётов Stripe.
