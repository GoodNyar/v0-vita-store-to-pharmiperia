# ADR-0002: Деньги — целые минорные единицы + код валюты

- **Статус:** принят
- **Дата:** 2026-07-03
- **Авторы:** Engineering (по итогам аудитов, отчёты 03–05)

## Контекст

Цены хранятся и считаются как `number` (float, EUR неявно): `lib/data.ts`, корзина, `actions/stripe.ts` (`Math.round(price * 100)` россыпью), `DECIMAL` в БД. Float-арифметика на деньгах даёт ошибки округления; неявная валюта блокирует мультивалютность (Phase 5 roadmap). Сейчас ~30 точек использования; через год их будут сотни.

## Решение

Единый тип `Money = { amount: number; currency: CurrencyCode }`, где `amount` — целое в минорных единицах (центы). Вся арифметика — в `lib/money.ts` (sum, multiply, format). В БД — `INTEGER`-колонки минорных единиц + `currency`. Конверсия в формат Stripe — в одном месте (`lib/stripe`). Форматирование — `formatMoney(money, locale)` вместо `formatEur`.

## Последствия

- Проще: точная арифметика; мультивалютность становится вопросом данных, а не рефакторинга; Stripe-суммы совпадают с доменными по определению.
- Сложнее: одноразовая миграция типов и колонок (шаг B4 blueprint); сброс формата localStorage-корзины (версия ключа).
- Затронут Playbook §1.7, §15.1.

## Альтернативы

- **Оставить float** — отклонено: классический источник багов округления и главный блокер мультивалютности.
- **Decimal-библиотека (big.js)** — отклонено: зависимость и сериализация сложнее, чем целые числа; для розничных цен целых минорных единиц достаточно.
- **Только UI-форматирование, хранение как есть** — отклонено: проблема в хранении и арифметике, не в отображении.
