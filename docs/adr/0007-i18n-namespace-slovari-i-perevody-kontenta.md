# ADR-0007: i18n — namespace-словари для UI, переводы контента в БД

- **Статус:** принят
- **Дата:** 2026-07-03
- **Авторы:** Engineering (по итогам аудитов, отчёты 03, 06)

## Контекст

Весь i18n — рукописный монолит `lib/i18n.tsx` (1787 строк): словарь RU/LV единым объектом в клиентском бандле каждой страницы, тип `Lang = "ru" | "lv"` зашит, плюрализации нет, переводы описаний товаров лежат в UI-словаре (`productDescriptions` по id), язык живёт в localStorage (см. ADR-0003 — источником становится URL). Третий язык (Phase 5: lt/et/en) при такой модели означает переписывание.

## Решение

1. **UI-строки** — статические словари по namespace и локали: `lib/i18n/dictionaries/{lv,ru}/{common,checkout,account,...}.ts`; загружаются на сервере по локали из URL, клиенту передаётся только нужный namespace.
2. **Контент** (названия, описания товаров, категории, блог) — в БД, в таблицах переводов (`*_translations(entity_id, locale, ...)`), не в код-словарях и не в колонках `_ru/_lv` для новых сущностей.
3. Локаль определяется URL-сегментом (ADR-0003); `Lang`-тип генерируется из конфига локалей — добавление языка = новая папка словарей + строка конфига.
4. API `t(key)` сохраняется на переходный период (дешёвая миграция вызовов); плюрализация — `Intl.PluralRules`.
5. Форматирование денег/дат — `Intl` по локали (`formatMoney`, ADR-0002), не захардкоженный `de-DE`.

## Последствия

- Проще: словари не раздувают бандл; третий язык — данные, а не рефакторинг; переводчик работает с БД/файлами, не с кодом.
- Сложнее: одноразовая нарезка монолита (скриптом по префиксам ключей); дисциплина «контент ≠ UI-строка».
- Затронут Playbook §4 (ключи по namespace), §15.6.

## Альтернативы

- **next-intl / paraglide** — приемлемо, но отклонено сейчас: собственная модель уже совместима по API, а внешняя библиотека — зависимость с собственным роутинг-мнением; пересмотр возможен при внедрении TMS (Phase 5) отдельным ADR.
- **Все переводы в БД (включая UI)** — отклонено: UI-строки версионируются вместе с кодом, который их рендерит; БД-словарь ломает эту связь.
- **Оставить монолит, дописывая языки** — отклонено: линейный рост бандла и объектных литералов; плюрализация не встраивается.
