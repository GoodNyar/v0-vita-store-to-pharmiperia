## What's in this RC

- `resolveMarketFromCookies()` — единый server-side market resolve (cookie `pharm_market` + headers)
- `createDraftOrder` — `orders.market_code`, per-market VAT (`vatRateBps`), DB shipping validation
- `createCheckoutSession` — `market_code` в Stripe metadata
- Checkout UI — server wrapper с DB shipping + market VAT display

### Unified pricing (Audit 22 M-1)

- `applyMarketPricingToCommerceProduct` — единый entry point для:
  - Catalog (`catalog-source.ts`)
  - Storefront API (`storefront.ts`)
  - Checkout / orders (`orders.ts`, `server-cart.ts`)

### DB-backed shipping (Audit 22 M-3)

- `getCheckoutShippingOptions(marketCode)` из `market_shipping_methods`
- Hardcoded shipping options удалены из checkout UI

### Release integrity (Audit 23 B-1)

## Known limitations (accepted for LV v1.0)

### B-2: ISR catalog vs per-market display pricing

Каталожные страницы (ISR, `revalidate=3600`) запекают LV-цены при prerender. Cookie-based market pricing в `catalog-source.ts` **не влияет** на отображаемую цену под ISR в runtime.

**Для LV-only v1.0 без price overrides — безвредно.**  
**Для LT/EE или per-market overrides — требует remediation до запуска.**

Подробности: [KNOWN_LIMITATIONS.md](KNOWN_LIMITATIONS.md#kl-1--per-market-display-pricing-vs-isr-b-2)

### Other deferred items

- LT/EE parcel stations в веб-checkout (LV stations hardcoded)
- Inventory reservations disabled by env flag
- CSP `unsafe-inline` not resolved
